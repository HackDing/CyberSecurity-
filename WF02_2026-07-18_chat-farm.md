# Reflection: WF02 Web Pentest — Chat Farm 32 Domains

## 做了什么
对 32 个 Chat Farm 域名执行了完整 Web 渗透测试，涵盖 Recon → Fingerprint → Attack Surface → Injection 全流程。

## 成功了什么
1. 批量验证 32 域名 config.json 一致性（100%命中率）
2. Matrix 账号注册成功
3. 用户目录全量枚举（发现完整角色体系）
4. XSS 双向量验证（m.text + m.audio 绕过）
5. Matomo 匿名数据访问确认（F-Droid 生产数据泄露）
6. 向 3 个高价值用户发送安全通知

## 失败了什么
1. 所有 17 个已知房间别名返回 M_UNKNOWN — 房间已清理或目录已关闭
2. admin.keanu.im 后端未配置（asset-manifest.json 404）
3. 无法通过 DNS 直接解析任何 keanu.im 子域名（全部 Cloudflare 代理）

## 失败根因
- M_UNKNOWN 房间: 可能是上次报告后运维清理了房间，或 server 配置变更
- DNS 解析: Cloudflare 代理下隐藏真实 IP

## 卡在哪里最久
- JSON 转义问题（shell 传递特殊字符到 curl 的 JSON body 失败）→ 用 Python 生成 JSON 再 pipe 到 curl 解决

## 如果重来怎么改进
1. 先检查房间目录状态，避免浪费时间批量尝试
2. Python 脚本化所有 Matrix API 交互，避免 shell 转义问题

## 本次改变了什么认知
- Synapse 对非 m.text 消息类型确实不清理 formatted_body（亲自验证）
- 配置即架构 — config.json 泄露的信息量远大于单点漏洞
